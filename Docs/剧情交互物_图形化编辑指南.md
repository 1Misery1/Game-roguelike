# 剧情交互物 · 图形化编辑指南

本指南说明如何在 Unity Editor 中可视化创建 / 编辑剧情交互物（如「封死的升降门」「熔炉控制台」），无需改 C# 代码即可调整对话、英雄分支、真相旗奖励、外观尺寸。

---

## 一、整体架构

```
StoryInteractableData (ScriptableObject)   ← 你编辑这个
        │
        │   被引用
        ▼
StoryInteractable (MonoBehaviour, 挂在场景物体上)
        │
        │   读取数据
        ▼
DialogueBox.Play(...)                       ← 自动播放对话
```

- **数据**：`Assets/Resources/Story/*.asset` —— 每个交互物对应一个 `.asset` 文件
- **载体**：`Assets/Scripts/Narrative/StoryInteractable.cs` —— 运行时根据数据自动配置外观与对话
- **触发**：`Assets/Scripts/Dev/GameBootstrap.cs` 中 `SpawnFloor1StoryDoor()` 等方法 —— 决定在哪个房间生成

## 二、入口在哪？三种典型操作

### A. 改对话内容（最常用）

1. Project 面板进入 `Assets/Resources/Story/`
2. 双击 `Floor1_SealedDoor.asset`（或其他剧情数据）
3. **Inspector 面板** 显示所有可编辑字段：
   - **身份**
     - `Object Id` —— 跨周目调查计数 ID
     - `Banner Text` —— 完成后顶部横幅文字
   - **外观（白盒占位）**
     - `Tint Color` —— 占位方块颜色
     - `Visual Scale` —— X/Y 缩放（世界单位）
     - `Collider Size` —— 触发盒大小
   - **对话分支**（核心）
     - 点击 `+` 添加一个 `Branch`
     - 每个 Branch 字段说明：
       | 字段 | 作用 |
       |---|---|
       | `Note` | 仅调试名，例如「首次-雷昂专属」 |
       | `Min Count` | 最少调查次数（**按本英雄**计数；换英雄重玩仍从 1 起） |
       | `Max Count` | 最多调查次数（按本英雄计数，0 = 不限） |
       | `Min Run Count` | 最少累计通关次数（多周目专用，0=不限） |
       | `Max Run Count` | 最多累计通关次数（0=不限） |
       | `Require Hero` | 仅此英雄触发（`Warrior`/`Ranger`/`Mage`/`Paladin`/`Hunter`；留空=任意） |
       | `Forbid Hero` | 排除此英雄（留空=不排除） |
       | `Lines` | 此分支贡献的对话行 |
     - 多个 Branch 会按列表顺序**拼接**（满足条件的全部追加） → 可用「旁白通用段 + 英雄专属段」组合
   - **玩家抉择（可选）**
     - `Choice Title` —— 弹窗顶部标题（留空 = 用 Banner Text）
     - `Choices` —— 选项列表（留空 = 不弹窗）。每个选项可独立配置：
       - `Label` —— 按钮显示文字
       - `Description` —— 选项下方描述（说明后果）
       - `Follow Lines` —— 选中后追加播放的对话行
       - `Run Story Flags` —— 选中后追加的本周目剧情旗（**用作"已选"标记**：选过后本周目不再重复弹窗）
       - `Truth Awards` —— 选中后追加的真相旗
       - `Grant Story Items` —— 选中后追加的剧情道具
       - `Add Corruption` —— 选中后的污染增量（叠加在主数据上）
       - `Banner Override` —— 选中后的横幅文字（覆盖主对话 banner）
     - 已有示例：冻结湖面（直视 / 打碎）、黑色镜子（对峙 / 转身）

   - **完成奖励**
     - `Run Story Flags` —— 本周目内剧情旗
     - `Truth Awards` —— 跨周目真相旗，每条字段：
       - `Flag` —— 旗 ID
       - `Require Hero` —— 专属英雄（首次调查即解锁；留空 = 任意英雄首次即解锁）
       - `Fallback Count` —— **按全局计数**：任意英雄顺序累计调查 N 次后无视 requireHero 解锁（建议 2-3，0 = 禁用后备）→ 保证任意英雄出场顺序都能凑齐 Crown 结局所需真相旗
       - ⚠️ **重要：** 给英雄专属真相旗设 `fallbackCount ≥ 2` 是单角色玩家能解锁全部结局的关键
     - `Grant Story Items` —— 本周目剧情道具（如「烧焦的工匠名册」）
     - `Add Corruption` —— 虚空污染增量（正=污染，负=净化）

### B. 对话占位符

在 `Speaker` / `Portrait Key` / `Text` 字段中可使用：

| 占位符 | 替换为 |
|---|---|
| `{hero}` | 当前英雄显示名（如「雷昂·铁誓」） |
| `{heroKey}` | 当前英雄美术键（如 `Warrior`，用于头像查找） |

**头像规则：** `Portrait Key` 留空 = 无头像（旁白）；写 `{heroKey}` = 自动用当前英雄头像。

### C. 创建一个**全新**剧情交互物（完全零代码）

1. Project 面板右键 → **Create → Game → Narrative → Story Interactable**
2. 命名（建议格式：`FloorN_名字.asset`），并放到 `Assets/Resources/Story/` 下
3. Inspector 填好：
   - `Object Id`（唯一）/ `Banner Text`
   - **生成位置：** `Spawn Floor`（1/2/3）+ `Spawn Room Index`（0 起）+ `Spawn Offset`（相对玩家出生点）
   - `Branches`（对话分支）
   - `Run Story Flags` / `Truth Awards` / `Grant Story Items` / `Add Corruption`
4. 不需要写任何 C# 代码——`GameBootstrap` 启动时通过 `Resources.LoadAll` 扫描 `Story/` 文件夹，按 `Spawn Floor` + `Spawn Room Index` 自动 spawn

如果 `Spawn Floor` 留 `0`，则不自动生成（用于需要代码触发的特殊交互物）。

## 四、常见问题

**Q：我添加了新 `.asset`，运行时找不到怎么办？**
A：检查文件是否在 `Assets/Resources/Story/` 目录下。`Resources.Load` 路径必须以此为根，且文件名不含扩展名。

**Q：同一个交互物想在不同英雄下显示完全不同的对话？**
A：建多个 `Branch`，分别设置 `Require Hero`。所有满足条件的分支会拼接播放。

**Q：想做「调查 3 次才触发隐藏对话」？**
A：在分支里设 `Min Count = 3`、`Max Count = 0`。当前调查次数由 `PersistentState.RecordInvestigation` 自动累计，跨周目保留。

**Q：横幅文字想根据英雄变化？**
A：当前 `Banner Text` 是固定字符串。如需英雄分支化，可扩展 SO 字段（告诉我即可）。
